# Database Reliability Expert Skill

Этот навык предназначен для обеспечения 100% надежности при изменении схемы базы данных (DDL). Его цель — предотвратить случайную потерю колонок, вьюшек или функций.

## Основные правила (Must-Follow)

### 1. Снимок перед изменениями (Snapshot Before Change)
Перед выполнением любого `DROP`, `ALTER` или `CREATE OR REPLACE`, вы ОБЯЗАНЫ сохранить текущее определение объекта.
- Используйте `pg_get_viewdef` для вьюшек.
- Используйте `pg_get_functiondef` для функций.
- Сохраняйте результат в `.agent/temp/snapshot_[object_name].sql`.

### 2. Аудит зависимостей (Dependency Audit)
Всегда проверяйте, какие объекты зависят от изменяемого объекта.
- Используйте скрипт `scripts/list_dependencies.sql`.
- Если вы используете `CASCADE`, вы ОБЯЗАНЫ явно перечислить все удаленные объекты и восстановить их в той же транзакции.

### 3. Проверка целостности (Post-Migration Verification)
После выполнения миграции ОБЯЗАТЕЛЬНО:
- Проверить наличие всех ожидаемых колонок во вьюшке (сравнить со снимком).
- Выполнить тестовый вызов обновленных RPC.
- Убедиться, что зависимые вьюшки не "сломались".

## Инструменты
- `view_file` для чтения снимков схемы.
- `mcp_supabase-mcp-server_execute_sql` для выполнения проверок.

## Алгоритм работы при обновлении View
1. Прочитать текущий DDL вьюшки.
2. Прочитать текущий список колонок.
3. Подготовить новый DDL, включив ВСЕ старые колонки + новые.
4. Выполнить миграцию.
5. Запросить `SELECT * FROM [view] LIMIT 1`, чтобы подтвердить работоспособность.
